前言

一个好的框架一定是在技术层面上进行了高度抽象封装和代码复用提炼的，从而最大程度减少编码工作量，提高开发效率，而一个好的编码工具旨在提示编码效率同时，同时指导用户怎样使用该框架，好的框架加好的编码工具方能将开发效率提示到极致。
说明:
1、适用范围:该工具是个轻量级的eclipse插件，适用eclipse 3.2以上版本。
2、生成的代码不受架构，各种框架等限制，生成任意文本类型的文件。
3、可视化的文档操作界面，非常灵活的选择想要生成的文档。
4、操作简单。
如何使用好该工具，重点就是要写一个好的模板和定义一个好的注释，模板是一次性编写的工作，写好后保存(可以只保存到本机，也可以保存到服务器供他人使用)，好的模板在工具可视化上体现为文件的目录结构明了，执行时有校验和提示向导，让用户一看就知道生成的内容是实现什么逻辑，生成相关文件需要满足的条件。

第一章:概述

代码生成原理:模板+数据=代码
设计驱动开发，这里的设计指的是用户写的java类(如实体类，接口类等)，而非UML设计文档。
简单讲就是一个简单的POJO类(或接口)，加上用户的业务注释，生成相关的文件代码，工具将用户的注释和POJO类的属性（或接口的方法）全解析成一个可视化的JSON数据（见【数据】章节），供模板编写调用。
模板采用freemarker语法编写，同时定义了一些模板编写规则，以及新增了一些模板标签标签（见【模板】章节）。

第三章:模板

代码生成中，模板创建是最关键的，一个模板可以生成一个文件，也可以生成多个文件，当然也可以一个都不生成(仅仅在弹出框生成个预览参考的代码)， 在模板中将这些都定义好后，工具会自动将模板转换成可视化的操作界面，选中自己想要的文件，一键生成即可，查看样例。
如何创建一个模板，点击查看。
模板一旦创建后，可以存放在本地，也可以存放到服务器共享给其他用户。用户可以使用本地模板，也可以直接使用服务器上的模板。
除了会使用freemarker语法外，还必须得掌握如下几个标签，如下这些标签非freemarker标签。
1、模板结构标签
2、文件生成标签
3、提示框标签
4、文档标签
5、文档追加标签
6、新增freemarker方法(V1.2上版本)。
  • filtMatch ：过滤出满足条件的节点，支持正则匹配。 
    例：遍历所有满足属性名为id,或name的所有属性节点。
 <#list filtMatch(sys.propList,"{\"propName\":\"id\"}","{\"propName\":\"name\"}") as e>
  ${e.propName}
 </#list>
例：遍历javaAnList值中含有Mapping关键字的节点。 
 <#list filtMatch(sys.propList,"{\"javaAnList\":\".*Mapping.*\"}") as e>  
   Match: ${e.propName}
 </#list>
  • filtNotMatch方法，过滤出不满足条件的节点，支持正则匹配。 
    例： 遍历所有满足属性名不为id 并且 也不为name的所有属性节点。
 <#list filtNotMatch(sys.propList,"{\"propName\":\"id\"}","{\"propName\":\"name\"}") as e>
   prop notMatch: ${e.propName}
 </#list>
